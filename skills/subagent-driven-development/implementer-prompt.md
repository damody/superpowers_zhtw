# 實現者子代理提示範本

在分派實現者子代理時使用此範本。

```
任務工具 (通用):
  description: "實現任務 N: [任務名稱]"
  prompt: |
    你正在實現任務 N: [任務名稱]

    ## 任務描述

    [從計劃中完整的任務文本 - 在此粘貼，不要讓子代理讀取文件]

    ## 背景

    [場景設置: 這適合於何處、依賴、架構背景]

    ## 開始之前

    如果你對以下內容有疑問:
    - 需求或驗收標準
    - 方法或實現策略
    - 依賴或假設
    - 任務描述中的任何不清楚之處

    **現在就提出。** 在開始工作前提出任何關切。

    ## 你的工作

    一旦你清楚了需求:
    1. 完全實現任務指定的內容
    2. 編寫測試 (如果任務要求，遵循 TDD)
    3. 驗證實現有效
    4. 提交你的工作
    5. 自我檢查 (見下文)
    6. 回報

    從以下位置工作: [目錄]

    **當你工作時:** 如果你遇到意外或不清楚的地方，**提出問題**。
    暫停和澄清總是可以的。不要猜測或做出假設。

    ## 報告前: 自我檢查

    用新鮮的眼光檢查你的工作。問自己:

    **完整性:**
    - 我是否完全實現了規格中的所有內容?
    - 我是否遺漏了任何需求?
    - 有沒有我沒有處理的邊界情況?

    **質量:**
    - 這是我最好的工作嗎?
    - 名字是否清晰和準確 (與事物做什麼匹配，而不是它們如何工作)?
    - 代碼是否乾淨且可維護?

    **紀律:**
    - 我是否避免了過度構建 (YAGNI)?
    - 我是否只構建了被要求的?
    - 我是否遵循了代碼庫中的現有模式?

    **測試:**
    - 測試是否真正驗證行為 (而不僅僅是 mock 行為)?
    - 如果需要，我是否遵循了 TDD?
    - 測試是否全面?

    如果在自我檢查期間發現問題，在報告前修正。

    ## 報告格式

    完成後，報告:
    - 你實現了什麼
    - 你測試了什麼和測試結果
    - 更改的文件
    - 自我檢查發現 (如有)
    - 任何問題或關切
```
