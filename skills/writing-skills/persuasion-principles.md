# 技能設計的說服原則

## 概述

LLM 對與人類相同的說服原則做出反應。理解這種心理學幫助你設計更有效的技能 - 不是為了操縱，而是為了確保即使在壓力下也遵循關鍵實踐。

**研究基礎:** Meincke 等人 (2025) 用 N=28,000 次 AI 對話測試了 7 個說服原則。說服技術將遵守率提高了一倍多 (33% → 72%, p < .001)。

## 七個原則

### 1. 權威
**它是什麼:** 對專業知識、證書或官方來源的尊重。

**它如何在技能中運作:**
- 命令性語言: "你必須"、"絕不"、"總是"
- 非談判的框架: "沒有例外"
- 消除決策疲勞和合理化

**何時使用:**
- 紀律執行的技能 (TDD、驗證要求)
- 安全關鍵實踐
- 既定最佳實踐

**示例:**
```markdown
✅ 先寫代碼？刪除它。重新開始。沒有例外。
❌ 在可行時考慮先寫測試。
```

### 2. 承諾
**它是什麼:** 與先前行動、陳述或公開聲明的一致性。

**它如何在技能中運作:**
- 需要宣布: "宣布技能使用"
- 強制明確選擇: "選擇 A、B 或 C"
- 使用跟踪: TodoWrite 用於檢查清單

**何時使用:**
- 確保技能實際被遵循
- 多步驟流程
- 問責機制

**示例:**
```markdown
✅ 找到技能時，你必須宣布: "我正在使用 [技能名稱]"
❌ 考慮讓你的夥伴知道你正在使用哪個技能。
```

### 3. 稀缺性
**它是什麼:** 來自時間限制或有限可用性的緊迫感。

**它如何在技能中運作:**
- 時間限制的要求: "在繼續前"
- 順序依賴: "立即在 X 後"
- 防止拖延

**何時使用:**
- 立即驗證要求
- 時間敏感的工作流程
- 防止 "我稍後做"

**示例:**
```markdown
✅ 完成任務後，立即在繼續前請求代碼審查。
❌ 你可以在方便時審查代碼。
```

### 4. 社會證明
**它是什麼:** 符合他人做的事或被認為是正常的事。

**它如何在技能中運作:**
- 普遍模式: "每次"、"總是"
- 失敗模式: "X 沒有 Y = 失敗"
- 建立規範

**何時使用:**
- 記錄普遍實踐
- 警告常見失敗
- 加強標準

**示例:**
```markdown
✅ 沒有 TodoWrite 跟踪的檢查清單 = 步驟被跳過。每次。
❌ 有些人發現 TodoWrite 對檢查清單有幫助。
```

### 5. 統一
**它是什麼:** 共享身份、"我們的感覺"、群體內歸屬。

**它如何在技能中運作:**
- 協作語言: "我們的代碼庫"、"我們是同事"
- 共享目標: "我們都想要質量"

**何時使用:**
- 協作工作流程
- 建立團隊文化
- 非等級制實踐

**示例:**
```markdown
✅ 我們是同事一起工作。我需要你的誠實技術判斷。
❌ 你應該可能告訴我我是否錯了。
```

### 6. 互惠
**它是什麼:** 返回收到的好處的義務。

**它如何運作:**
- 謹慎使用 - 可能感到操縱
- 在技能中很少需要

**何時避免:**
- 幾乎總是 (其他原則更有效)

### 7. 喜歡
**它是什麼:** 與我們喜歡的人合作的偏好。

**它如何運作:**
- **不要用於遵守**
- 與誠實的反饋文化相衝突
- 創造諂媚

**何時避免:**
- 對於紀律執行總是

## 按技能類型的原則組合

| 技能類型 | 使用 | 避免 |
|---------|------|------|
| 紀律執行 | 權威 + 承諾 + 社會證明 | 喜歡、互惠 |
| 指導/技術 | 適度權威 + 統一 | 重權威 |
| 協作 | 統一 + 承諾 | 權威、喜歡 |
| 參考 | 僅清晰 | 所有說服 |

## 為什麼這有效: 心理學

**明亮線規則減少合理化:**
- "你必須" 消除決策疲勞
- 絕對語言消除 "這是個例外嗎?" 的問題
- 明確的反合理化計數器特定的漏洞

**實現意圖創建自動行為:**
- 清晰的觸發器 + 必需的行動 = 自動執行
- "當 X，做 Y" 比 "通常做 Y" 更有效
- 減少遵守的認知負荷

**LLM 是準人類:**
- 在包含這些模式的人類文本上培訓
- 權威語言在訓練數據中排在遵守前面
- 承諾序列 (聲明 → 行動) 經常被模擬
- 社會證明模式 (每個人都做 X) 建立規範

## 倫理使用

**合法:**
- 確保遵循關鍵實踐
- 創建有效的文檔
- 防止可預測的失敗

**非法:**
- 為個人收益而操縱
- 創建虛假的緊迫感
- 基於內疚的遵守

**測試:** 如果此技術在他們充分理解情況下為用戶的真正利益服務，會合法嗎?

## 快速參考

設計技能時，問:

1. **它是什麼類型?** (紀律 vs 指導 vs 參考)
2. **我試圖改變什麼行為?**
3. **哪個原則適用?** (通常對紀律使用權威 + 承諾)
4. **我是否合併了太多?** (不要使用全部七個)
5. **這是道德的嗎?** (為用戶的真正利益服務?)
