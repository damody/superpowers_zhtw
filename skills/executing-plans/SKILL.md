# 執行計劃

## 概述

加載計劃，批判性審查，分批執行任務，在批次之間報告以供審查。

**核心原則：** 帶檢查點的分批執行，供架構師審查。

**開始時宣告：** 「我正在使用 executing-plans 技能實作此計劃。」

## 流程

### 步驟 1：加載和審查計劃
1. 閱讀計劃文件
2. 批判性審查 - 識別關於計劃的任何問題或疑慮
3. 如果有疑慮：在開始之前向您的人類夥伴提出
4. 如果沒有疑慮：創建 TodoWrite 並繼續

### 步驟 2：執行批次
**預設：前 3 個任務**

對於每個任務：
1. 標記為 in_progress
2. 完全遵循每個步驟（計劃有小步驟）
3. 按指定運行驗證
4. 標記為已完成

### 步驟 3：報告
批次完成時：
- 顯示已實作的內容
- 顯示驗證輸出
- 說：「準備好接受反饋。」

### 步驟 4：繼續
根據反饋：
- 如果需要應用更改
- 執行下一批次
- 重複直到完成

### 步驟 5：完成開發

所有任務完成並驗證後：
- 宣告：「我正在使用 finishing-a-development-branch 技能完成此工作。」
- **必需的子技能：** 使用 superpowers:finishing-a-development-branch
- 遵循該技能驗證測試、呈現選項、執行選擇

## 何時停止並尋求幫助

**立即停止執行當：**
- 在批次中遇到阻礙（缺少依賴項、測試失敗、指令不清楚）
- 計劃有關鍵缺口阻止開始
- 您不理解某個指令
- 驗證反覆失敗

**尋求澄清而不是猜測。**

## 何時重新審視早期步驟

**返回審查（步驟 1）當：**
- 夥伴根據您的反饋更新計劃
- 基本方法需要重新思考

**不要強行通過阻礙** - 停止並詢問。

## 記住
- 首先批判性審查計劃
- 完全遵循計劃步驟
- 不要跳過驗證
- 當計劃要求時引用技能
- 批次之間：僅報告並等待
- 遇到阻礙時停止，不要猜測
